## HTML 일정 파일 제작 규칙 (Day별 파일 공통)

### 탭 구조
- **기본 3탭:** 📋 일정 / 🍽 식당 / 🛒 마트
- 점심·저녁 구분이 있는 날: 📋 일정 / 🍽 점심 / 🌙 저녁 / 🛒 마트
- **이동일 포함 마트 탭 항상 유지** (도착 후 마트 방문 가능)
- 이동일 패스권+교통권 정보: 일정 탭 **최상단 고정 카드**로 배치 (드로어 방식 폐기)

### 식당 카드 구조
- 와이프 저장 목록 먼저, 그 다음 AI 추천 (배지로 구분)
- 저장된 곳: 원본 메모 💬 그대로 노출
- AI 추천: 💡 card-tip으로 동선/타이밍 코멘트
- **추천 개수:** 와이프 저장이 적거나 없는 지역은 AI 추천을 넉넉하게 (종류 겹쳐도 OK)
- **동선 범위:** 관광지 코앞만 아니라 동선 중간에 들를 수 있는 곳도 포함

### 식당 선택 기능 (클릭 시 4가지 동시 처리)
1. 지도에 식당 핀 추가
2. 기존 동선 숨김 → 식당 경유 동선으로 교체
3. 일정 탭 식사 항목에 식당명/설명 채워짐
4. **그 다음 이동카드도 함께 표시** ← 매번 체크 필수
- 재클릭 시 전부 원상복구
- 점심에 선택된 식당은 저녁 탭에 "🍽 점심에 선택됨" 크로스배지 표시 (반대도 동일)

### 카페 드로어 (2개 분리)
- **☕ 오전 카페:** 오전 관광지 동선 기준, 오전 일정 항목 사이에 버튼 배치
- **☕ 오후 카페:** 오후 관광지 동선 기준, 오후 일정 항목 사이에 버튼 배치
- 점심 식당 선택 후: 카드 순서는 유지, 카드 내 거리 뱃지만 "선택한 식당에서 도보 ○분"으로 업데이트
- 카페 카드마다 "📍 ○○ 근처 도보 ○분" 위치 명시
- 카페 버튼 클릭 → 드로어 펼쳐짐 + 지도에 ☕ 핀 추가

### 마트 탭 구성
- **현지 마트** (SPAR, EUROSPAR, Kaufland, Billa 등) + 영업시간 표기
- **한인마트 또는 한식 식재료 취급점** (라면·햇반·김치 구입 가능한 곳)
- **한식당** (도시 내 또는 동선 근처)
- 없으면: "해당 지역 없음" 명시
- 멀면: 거리·소요시간 명시 후 포함 여부 판단

### 이동일 교통 카드
- 일정 탭 최상단 고정, 헤더 클릭 시 접기/펴기 (▾ 아이콘)
- id: `ttc-body` (바디), `ttc-chevron` (아이콘)
- 이전 도시 출발지 핀 + 현재 도시 도착지 핀 + 도시 간 점선 셔틀 동선 표시

### 지도 핀/동선
- 물방울형 divIcon, 번호+이름 라벨
- 도보=주황 점선, 대중교통=네이비 실선, 셔틀=연파랑(`#7A9ABE`) 점선, 동선 끝 SVG 삼각형 화살표 16px
- 초기 줌: fitBounds (그날 핵심 핀 자동 포함)
- 이동일: 출발·도착 도시 핀 모두 포함하여 fitBounds
- **지도 이동:** `map.panTo()` 사용 — `map.setView(target, 16)` 금지 (줌 레벨 유지)
- 핀 클릭 → 일정 탭 해당 항목 자동 스크롤 + 하이라이트
- **동선 교체 시 기본 화살표 처리:** 기본 동선 라인 반드시 변수로 저장 → 식당 선택 시 `setStyle({opacity:0})` + `_arrow.setOpacity(0)` → 재클릭 시 복원(`opacity:.78`)

### 교통 거점 핀 규칙
- 버스 정류장·케이블카 탑승장·기차역 등 **이동 수단 탑승 위치도 지도에 핀 표시**
- 교통 거점 핀 스타일: 🚌/🚡/🚂 이모지 + 장소명 라벨, 색상 `#7A9ABE`
- 이동카드(`.move-card`) 내에 해당 거점 구글지도 링크 추가
- **필수 거점 목록:** 아래 "교통 거점 위치 목록" 섹션 참조

### 구글지도 연동 규칙 (전체 공통)
- **모든 장소 카드(관광지·식당·카페·마트·교통거점)에 구글지도 링크 필수**
- 링크 형식: 좌표 우선 `https://maps.google.com/?q=위도,경도`, 없으면 `?q=장소명+도시명`
- 카드 내 버튼 형태: `<a class="gmap-btn" href="..." target="_blank">📍 구글지도</a>`
- 일정 탭 타임라인 `.tl-map-icon` 📍: 구글지도 링크 연결 (기존 구현 유지)
- 마트·카페 카드: 카드 하단에 `gmap-btn` 버튼 별도 추가
- Leaflet 핀 클릭 = 일정 스크롤 / 구글지도 링크 = 외부 앱 열기 (역할 분리)
- `gmap-btn` CSS: `display:inline-block; font-size:11px; color:var(--p-blue); text-decoration:none; padding:4px 10px; border:1px solid var(--border); border-radius:var(--radius-badge); margin-top:6px;`

### 마트 탭 추가 규칙
- 마트 카드 클릭 → Leaflet 지도 핀 토글 (on-map 클래스, 재클릭 시 핀 제거)
- 마트 카드에 구글지도 링크 (`gmap-btn`) 필수

### 잘츠부르크 카드 — 개별 구매 원칙, 단 조건부 검토 가능

**Guest Mobility Ticket 제공 전제 시 → 개별 구매가 유리**
- 대중교통은 Guest Mobility Ticket으로 이미 무료 → 카드의 교통 가치 소멸
- 관광지 개별 요금 합계: 헬브룬 €16.50 + 호엔잘츠부르크 €15.50 + 엘리베이터 €4.80 = **€36.80**
- 잘츠부르크 카드 24h €38 vs 개별 €36.80 → 카드가 €1.20 더 비쌈
- 개별 온라인 예매로 줄서기 해결 가능
- **결론: 개별 구매**

**만약 빌라 엘리사에서 Guest Mobility Ticket을 제공하지 않는다면 → 카드 재검토**
- 대중교통 7/29 하루 버스 이용분(3~4회 × €2.50) = 약 €7.50~€10
- 개별 합계 €36.80 + 버스 €10 = **€46.80**
- 잘츠부르크 카드 24h **€38** → 이 경우 카드가 **€8.80 저렴**
- 숙소 체크인 시 Guest Mobility Ticket 발급 여부 즉시 확인 → 미발급 시 카드 구매 고려
